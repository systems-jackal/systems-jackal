## About

I'm a Computer Security & Forensics student who builds and maintains backend systems and infrastructure. My day-to-day involves writing backend services in **Java and Rust**, managing Linux servers, keeping networks stable and locked down, and making sure security is woven into the process rather than added at the end.

I founded **Unity Bridge** to do this practically — providing small businesses and communities in Kenya with reliable, secure connectivity and IT support. Most of what I know came from having to solve real problems, not just study them.

## Skills & Tools

I work across several layers of a system. Here's an honest look at where I am with each:

### Java — Backend Development
Java is my primary backend language and where I have the most depth. I build production-grade REST APIs and microservices using **Spring Boot**, manage dependencies and builds with **Maven**, and handle data persistence with **Spring Data JPA** and **Hibernate**. I structure projects around clean architecture — proper layering, separation of concerns, and code that's easy to read and audit. I also apply **Spring Security** to lock down endpoints and handle authentication the right way, not as an afterthought.

**Java ecosystem I work with:**
- Spring Boot · Spring MVC · Spring Security · Spring Data JPA
- Hibernate ORM · Maven · REST API design
- JUnit for testing · Lombok · environment-based configuration

### Rust — Backend Development
I use Rust for backend work where performance and memory safety matter — lightweight services, CLI tools, and anything that needs to stay lean and reliable under load. I work with **Actix-web** for HTTP services and **Tokio** for async runtimes. Rust's ownership model changes how you think about code, and that discipline carries over into everything else I write. It complements Java well: Java for rich, framework-heavy services; Rust for fast, tight, low-overhead components.

**Rust ecosystem I work with:**
- Actix-web · Tokio · Serde · Cargo
- Async/await patterns · CLI tooling · systems-level backend services

### Server Management
I manage Linux servers — Ubuntu primarily — handling setup, hardening, user access, services, and ongoing maintenance. I run Nginx as a reverse proxy, manage TLS certificates, and use Cloudflare for DNS and an extra layer of protection. This is hands-on, practical work I do regularly through Unity Bridge.

### Networking
I understand networking at a deeper level than most backend developers — subnetting, routing, firewall rules, VPN configuration. Through Unity Bridge I've had to design and maintain small network infrastructure for real clients. I'm also exploring IoT security, which sits at the edge of networking and embedded systems.

### Secure Development Operations
I integrate security into how I build, not just what I build — managing secrets properly, running static analysis, and keeping CI/CD pipelines clean. I use GitHub Actions for automation and Git for everything. This is an area I'm actively developing further.
